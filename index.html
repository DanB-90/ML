1. Introducción al Machine Learning (ML)

El Machine Learning (Aprendizaje Automático) es una rama de la inteligencia artificial que permite a los sistemas aprender patrones a partir de datos sin ser programados explícitamente. En esta sección exploraremos los conceptos básicos que sustentan esta tecnología revolucionaria.

2. Tipos de aprendizaje

En Machine Learning existen diferentes paradigmas de aprendizaje según la naturaleza de los datos y los problemas a resolver. El aprendizaje supervisado utiliza datos etiquetados para entrenar modelos predictivos, mientras que el no supervisado descubre patrones ocultos en datos sin etiquetar. También exploraremos el aprendizaje por refuerzo, donde los agentes aprenden mediante interacción con su entorno. Cada enfoque tiene sus fortalezas y aplicaciones específicas en la industria.

3. Modelos

Un modelo es una construcción conceptual que, como un mapa o una ecuación, reconstruye la realidad para entenderla mejor y usarla a nuestro favor. Los datos, que a menudo son multidimensionales, nos permiten construir estos modelos y los parámetros ajustan su flexibilidad para adaptarse a la evidencia

4. Modelos y algoritmos en ML

Es común referirse a los términos algoritmos de Machine Learning y modelos de Machine Learning como si tuvieran el mismo significado, pero son conceptos diferentes aunque relacionados. Un algoritmo y un modelo son necesarios para resolver un problema de Machine Learning. Un algoritmo de Machine Learning es esencialmente una serie de pasos que se programan en un computador y que se repiten de forma iterativa. Su propósito principal es encontrar los parámetros más adecuados para el modelo que se está desarrollando. Este proceso a menudo comienza asignando valores iniciales aleatorios a los parámetros y luego refina estos valores progresivamente basándose en la comparación de las predicciones generadas con los resultados reales, reduciendo el error a lo largo de múltiples iteraciones. Por otro lado, un modelo de Machine Learning es el producto final de aplicar un algoritmo. Una vez que el algoritmo ha completado su trabajo (por ejemplo, después de evaluar todas las iteraciones necesarias), el resultado es el modelo. Este modelo es lo que se utiliza para tomar nuevos datos y generar predicciones o responder al problema que se está intentando resolver. En un ejemplo simple, el modelo podría estar compuesto por la ecuación de una línea recta con los parámetros finales encontrados por el algoritmo, junto con el criterio para clasificar los datos de entrada (por ejemplo, si el resultado de la ecuación es positivo o negativo).

6. Ejemplo práctico: clasificación usando KNN

Objetivo

Implementar un modelo de Machine Learning para clasificar especies de flores Iris basado en características morfológicas.

Contexto

El dataset Iris es un conjunto de datos clásico que contiene medidas de sépalos y pétalos de tres especies de flores Iris (setosa, versicolor y virginica). Tu tarea es construir un modelo predictivo que aprenda a identificar la especie correcta a partir de las mediciones.

Pasos a realizar

  1. Carga y Exploración de Datos
    • Cargar el dataset Iris usando scikit-learn.
    • Identificar las características (features) y la variable objetivo (target).
  2. Preprocesamiento
    • Dividir los datos en conjuntos de entrenamiento (70%) y prueba (30%).
    • Estandarizar las características usando StandardScaler.
  3. Modelado
    • Entrenar un clasificador KNN con n_neighbors=3.
    • Realizar predicciones sobre el conjunto de prueba.
  4. Evaluación
    • Calcular la precisión (accuracy) del modelo.
    • Reportar el porcentaje de aciertos.

Código


# Importar librerías necesarias
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsClassifier
from sklearn.metrics import accuracy_score

# Cargar dataset
iris = load_iris()
X = iris.data  # Características (4 medidas de las flores)
y = iris.target  # Variable objetivo (especies: 0, 1, 2)

# Dividir datos en entrenamiento y prueba
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# Escalar características (importante para muchos algoritmos)
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test = scaler.transform(X_test)

# Crear y entrenar modelo (K-Vecinos más Cercanos)
modelo = KNeighborsClassifier(n_neighbors=3)
modelo.fit(X_train, y_train)

# Hacer predicciones
predicciones = modelo.predict(X_test)

# Evaluar rendimiento
precision = accuracy_score(y_test, predicciones)
print(f"Precisión del modelo: {precision:.2%}")

7. Retos Importantes en ML

1. Retos en Datos

a. Calidad de Datos

Problema: Datos incompletos, ruidosos o sesgados ("Garbage In, Garbage Out").

Ejemplo: Dataset de reconocimiento facial con predominio de un grupo étnico.

Soluciones:
  • Técnicas de imputación (KNN, regresión)
  • Análisis de distribuciones (SHAP, LIME)
  • Generación de datos sintéticos (GANs, SMOTE)

b. Escasez de Datos

Problema: En dominios como medicina o manufactura, obtener datos etiquetados es costoso.

Ejemplo: Diagnóstico de enfermedades raras con pocos casos registrados.

Soluciones:
  • Transfer Learning (reutilizar modelos preentrenados)
  • Data Augmentation (rotaciones, transformaciones en imágenes)
  • Active Learning (selección estratégica de muestras a etiquetar)

2. Retos en Modelado

a. Overfitting vs. Underfitting

Overfitting: Modelo memoriza datos de entrenamiento pero no generaliza.

Ejemplo: Árbol de decisión con profundidad excesiva.

Soluciones:
  • Regularización (L1/L2)
  • Dropout (en redes neuronales)
  • Early Stopping

Underfitting: Modelo demasiado simple para capturar patrones.

Ejemplo: Regresión lineal aplicada a datos no lineales.

Soluciones:
  • Aumentar complejidad del modelo
  • Feature Engineering

b. Compensación Sesgo-Varianza

Dilema: Reducir el error por sesgo (underfitting) aumenta la varianza (overfitting).

Enfoque óptimo:
  • Modelos que minimizan el error generalizado
  • Ejemplo: Grid Search para hiperparámetros

3. Retos en Ética y Justicia

a. Sesgos Algorítmicos

Problema: Modelos que replican/precipitan discriminación.

Ejemplo: Sistemas de reclutamiento que favorecen un género.

Soluciones:
  • Auditorías de fairness (métricas: igualdad de oportunidades, paridad demográfica)
  • Técnicas de debiasing (reducción de correlaciones espurias)

b. Explicabilidad

Problema: Modelos "caja negra" (Ej: deep learning) dificultan la transparencia.

Soluciones:
  • SHAP values, LIME (explicaciones locales)
  • Uso de modelos intrínsecamente interpretables (árboles, regresiones)

4. Retos Operacionales

a. Despliegue en Producción (MLOps)

Problema: El "valle de la muerte" entre el prototipo y la implementación real.

Ejemplo: Modelo funciona en Jupyter pero falla en APIs en tiempo real.

Soluciones:
  • Contenedores (Docker), orquestación (Kubernetes)
  • Monitoreo continuo (deriva de datos, métricas en tiempo real)

b. Escalabilidad

Problema: Modelos que no soportan grandes volúmenes de datos o usuarios.

Soluciones:
  • Optimización con ONNX, cuantización de modelos
  • Inferencia en edge devices (TF Lite, Core ML)

5. Retos Emergentes

a. Privacidad

Problema: Riesgo de filtración de datos sensibles durante el entrenamiento.

Ejemplo: Ataques de membership inference.

Soluciones:
  • Differential Privacy (añadir ruido controlado a los datos)
  • Federated Learning (entrenamiento descentralizado)

b. Sostenibilidad

Problema: Huella de carbono de modelos grandes (Ej: GPT-3 ≈ 190 ton CO₂).

Soluciones:
  • Modelos eficientes (TinyML, arquitecturas Sparse)
  • Green AI (selección de hardware eficiente)
