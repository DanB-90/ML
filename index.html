6. Ejemplo pr√°ctico: clasificaci√≥n usando KNN

Objetivo

Implementar un modelo de Machine Learning para clasificar especies de flores Iris basado en caracter√≠sticas morfol√≥gicas.

Contexto

El dataset Iris es un conjunto de datos cl√°sico que contiene medidas de s√©palos y p√©talos de tres especies de flores Iris (setosa, versicolor y virginica). Tu tarea es construir un modelo predictivo que aprenda a identificar la especie correcta a partir de las mediciones.

Pasos a realizar

  1. Carga y Exploraci√≥n de Datos
    • Cargar el dataset Iris usando scikit-learn.
    • Identificar las caracter√≠sticas (features) y la variable objetivo (target).
  2. Preprocesamiento
    • Dividir los datos en conjuntos de entrenamiento (70%) y prueba (30%).
    • Estandarizar las caracter√≠sticas usando StandardScaler.
  3. Modelado
    • Entrenar un clasificador KNN con n_neighbors=3.
    • Realizar predicciones sobre el conjunto de prueba.
  4. Evaluaci√≥n
    • Calcular la precisi√≥n (accuracy) del modelo.
    • Reportar el porcentaje de aciertos.

C√≥digo


# Importar librer√≠as necesarias

from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import KNeighborsClassifier
from sklearn.metrics import accuracy_score

# Cargar dataset
iris = load_iris()
X = iris.data  # Caracter√≠sticas (4 medidas de las flores)
y = iris.target  # Variable objetivo (especies: 0, 1, 2)

# Dividir datos en entrenamiento y prueba
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# Escalar caracter√≠sticas (importante para muchos algoritmos)
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test = scaler.transform(X_test)

# Crear y entrenar modelo (K-Vecinos m√°s Cercanos)
modelo = KNeighborsClassifier(n_neighbors=3)
modelo.fit(X_train, y_train)

# Hacer predicciones
predicciones = modelo.predict(X_test)

# Evaluar rendimiento
precision = accuracy_score(y_test, predicciones)
print(f"Precisi√≥n del modelo: {precision:.2%}")

1. Retos en Datos
